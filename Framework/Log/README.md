# Unity Console 日志输出控制器

## 功能概述

这个日志控制器允许你按文件目录或单个文件来控制Unity Console中的日志输出。支持精细化的日志管理，配置会自动保存到本地文件。

## 主要特性

✅ **目录级控制** - 可以按文件夹路径开启/关闭日志输出  
✅ **文件级精细控制** - 最小粒度到单个文件  
✅ **默认关闭策略** - 所有日志默认不输出，需要用户主动开启  
✅ **配置持久化** - 设置保存到本地文件，重启Unity后保持  
✅ **无感使用** - 继续使用Unity原生的Debug.Log函数  
✅ **可视化界面** - 提供Editor窗口进行配置管理  

## 快速开始

### 1. 打开日志控制器窗口

在Unity菜单栏中选择：`Tools > Log Controller`

### 2. 配置日志输出

- **启用目录**: 点击目录旁的"启用"按钮，该目录下所有文件的日志都会输出
- **禁用目录**: 点击目录旁的"禁用"按钮，该目录下所有文件的日志都不会输出
- **文件级控制**: 可以单独控制某个文件的日志输出，会覆盖目录设置

### 3. 使用日志

在代码中正常使用Unity的Debug.Log函数：

```csharp
using UnityEngine;

public class MyScript : MonoBehaviour
{
    void Start()
    {
        Debug.Log("这条日志会根据配置决定是否显示");
        Debug.LogWarning("警告日志");
        Debug.LogError("错误日志");
    }
}
```

## 配置说明

### 默认状态

- **默认关闭**: 所有日志默认不输出
- **手动开启**: 需要在Editor窗口中手动启用需要的目录或文件

### 配置优先级

1. **文件级设置** > **目录级设置** > **默认设置**
2. **禁用设置** > **启用设置**（同级别下）

### 配置示例

假设你有以下目录结构：
```
Assets/Scripts/
├── Framework/
│   ├── UI/
│   │   ├── GameButton.cs
│   │   └── GameText.cs
│   └── Log/
│       └── LogController.cs
└── Game/
    └── Player.cs
```

配置示例：
- 启用 `Framework/UI/` 目录 → `GameButton.cs` 和 `GameText.cs` 的日志都会显示
- 禁用 `GameButton.cs` 文件 → 只有 `GameText.cs` 的日志会显示
- 启用 `Game/` 目录 → `Player.cs` 的日志会显示

## Editor窗口功能

### 工具栏功能

- **搜索**: 按文件名搜索
- **仅显示已配置**: 只显示有明确配置的项目
- **全部展开/折叠**: 控制文件树的展开状态
- **刷新**: 重新扫描项目文件
- **全部启用/禁用**: 批量操作
- **重置配置**: 清除所有配置
- **保存配置**: 手动保存配置到本地

### 状态指示

- **绿色"启用"**: 该路径的日志会输出
- **红色"禁用"**: 该路径的日志不会输出  
- **灰色"继承"**: 继承父目录的设置

### 操作按钮

- **启用**: 启用该路径的日志输出
- **禁用**: 禁用该路径的日志输出
- **重置**: 移除该路径的配置，恢复继承状态

## 配置文件

配置文件保存在：`Assets/Scripts/Framework/Log/LogConfig.json`

配置文件格式：
```json
{
  "version": "1.0",
  "defaultEnabled": false,
  "enabledPaths": [
    "d:/karma4/client/karma_dev/assets/scripts/framework/ui"
  ],
  "disabledPaths": [
    "d:/karma4/client/karma_dev/assets/scripts/framework/ui/gamebutton.cs"
  ]
}
```

## 性能说明

- 使用缓存机制，避免重复的路径匹配计算
- 调用栈解析经过优化，性能影响最小
- 配置更改时会自动清除缓存，确保配置立即生效

## 注意事项

1. **LogController自身的日志**: 带有`[LogController]`标记的日志不会被过滤，确保系统正常运行
2. **异常日志**: 异常日志通常会显示，除非明确禁用
3. **路径格式**: 内部使用统一的小写路径格式，确保跨平台兼容
4. **配置保存**: 配置会在应用退出时自动保存，也可以手动保存

## 故障排除

### 日志控制器未工作

1. 检查Console中是否有`[LogController]`相关的错误信息
2. 确认配置文件是否正确加载
3. 在Editor窗口中检查配置状态

### 配置不生效

1. 点击Editor窗口中的"刷新"按钮
2. 检查路径配置是否正确
3. 确认文件路径的大小写和格式

### 性能问题

1. 避免在高频调用的代码中使用大量Debug.Log
2. 定期清理不需要的配置项
3. 使用"仅显示已配置"选项减少Editor窗口的显示项目
